serilaizer添加自定义字段

Posted on 2018-12-26 | In Django

在使用django restframework serializer 序列化在django中定义的model时，有时候我们需要额外在serializer中增加一些model中没有的字段。有两种方法实现这个目的。

假设现在有一个Animal模型，其中有name, type, country字段，country为外键。我们在序列化Animal时，需要额外增加外键country的area信息。

方法一：

1.首先，在django的model中定义property

2.然后在定义django restframework的serializer时，使用字段的source参数指向我们上面定义好的property

方法二：

不修改django的model，直接使用SerializerMethodField(method_name=None)字段

几点说明：

  1. SerializerMethodFiel是一个read-only字段
  2. 当不指定其method_name时，默认为get_<field_name>
  3. 如果使用ModelSerializer并指定字段时，要包含此时定义的字段

Mysql数据库操作

Posted on 2018-12-07 | In 数据库

​ Mysql数据库操作：

一， Mysql数据库

  1. 创建数据库：CREATE DATABSE IF NOT EXISTS databasename charset utf8;
  2. 删除数据库：DROP DATABASE databasename;
  3. 选取数据库：USE databasename;
  4. 关键字：NULL （数据列可包含空值）NOT NULL ， DEFAULT(默认值)，PRIMARY KEY（主键） AUTO_INCREMENT（自动递增，用于整形）CHARACTER SET name（指定一个字符集，比如：utf8）
  5. 建表：CREATE TABLE table_name （colnum_name colnum_type）ENGINE=InnoDB DEFAULT CHARSET=utf8;(指定引擎和字符集)
  6. 删除表：
    1. DROP TABLE table_name;(:删除表全部数据和表结构，立刻释放磁盘空间，不管是 Innodb 和 MyISAM )
    2. TRUNCATE TABLE table_name;(删除表全部数据，保留表结构会立即释放磁盘空间)
    3. DELETE FROM table_name;(删除全部表数据，保留表结构，innodb)
